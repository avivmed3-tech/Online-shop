# Online-shop

קטלוג מוצרים **ציבורי + B2B** — כל גולש רואה את הקטלוג המלא במחירי המחירון בלי התחברות; לקוחות עסקיים (שבעל העסק פותח להם חשבון) מתחברים ורואים את המחירים האישיים שלהם (אחוז הנחה ו/או מחיר מותאם פר-מוצר), מבצעים הזמנות שנשמרות, ובעל העסק מנהל מוצרים, לקוחות, מחירונים והזמנות.

**קישור לאתר:** https://avivmed3-tech.github.io/Online-shop/

## ארכיטקטורה

- **Frontend:** `catalog.html` — אפליקציית React (קובץ יחיד, PWA) שמתארחת ב-GitHub Pages.
- **Backend:** Supabase (Postgres + Auth + Edge Functions), פרויקט `online-shop-b2b`.

### תפקידים
- **גולש אנונימי**: רואה את כל הקטלוג במחירי מחירון, יכול לשלוח הזמנה בוואטסאפ (לא נשמרת במערכת).
- **admin** (בעל העסק): מנהל מוצרים, פותח חשבונות לקוחות, קובע אחוז הנחה **ומחירון אישי פר-מוצר** לכל לקוח, וצופה בכל ההזמנות.
- **customer** (לקוח עסקי): רואה קטלוג עם המחיר האישי שלו, מזמין, וצופה בהיסטוריית ההזמנות שלו.

### תמחור ללקוח עסקי
1. אם הוקצה **מחיר מותאם** למוצר (`customer_prices`) — הוא הקובע.
2. אחרת — `base_price` × (1 − `discount_pct`).

### אבטחה
- **RLS** על כל הטבלאות — אנונימי רואה רק מוצרים פעילים; לקוח רואה רק את הנתונים שלו; אדמין רואה הכל.
- **תמחור בצד השרת** — מחיר היחידה בהזמנה מחושב ב-trigger (מחיר מותאם אם קיים, אחרת `base_price` × (1 − הנחה)), כך שלא ניתן לזייף מחיר מהדפדפן.
- יצירת חשבונות לקוח נעשית דרך Edge Function (`create-customer`) המאמתת שהקורא הוא אדמין ומשתמשת ב-service-role בצד השרת בלבד.

## מבנה
```
catalog.html                              # האפליקציה (React + Supabase JS)
supabase/migrations/0001_b2b_schema.sql   # סכמה + RLS + טריגרים
supabase/migrations/0002_public_catalog_custom_prices.sql  # קטלוג ציבורי + מחירון אישי
supabase/functions/create-customer/       # פתיחת לקוח (admin בלבד)
```

## הגדרה
ב-`catalog.html`, בתוך `CONFIG`: `supabaseUrl`, `supabaseAnonKey`, ו-`whatsappNumber` (מספר העסק לקבלת הזמנות).
